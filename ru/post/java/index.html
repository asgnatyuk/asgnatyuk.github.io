Java

Java - это высокоуровневый объектно-ориентированный язык программирования, основанный на классах, который разработан так, чтобы иметь как можно меньше зависимостей от реализации. Это язык программирования общего назначения, который позволяет программистам писать один раз, выполнять везде (WORA), что означает, что скомпилированный Java-код может работать на всех платформах, поддерживающих Java, без необходимости перекомпиляции. Java-приложения обычно компилируются в байткод, который может работать на любой виртуальной машине Java (JVM) независимо от архитектуры компьютера. Синтаксис Java похож на синтаксис C и C++, но имеет меньше низкоуровневых средств, чем любой из них. Время выполнения Java предоставляет динамические возможности (такие как отражение и модификация кода во время выполнения), которые обычно недоступны в традиционных компилируемых языках. По данным GitHub на 2019 год, Java является одним из самых популярных языков программирования, особенно для клиент-серверных веб-приложений, с 9 миллионами разработчиков.

Java была первоначально разработана Джеймсом Гослингом в компании Sun Microsystems и выпущена в мае 1995 года как основной компонент платформы Java компании Sun Microsystems. Оригинальный и эталонный компиляторы Java, виртуальные машины и библиотеки классов были первоначально выпущены компанией Sun под проприетарными лицензиями. По состоянию на май 2007 года, в соответствии со спецификациями Java Community Process, Sun перелицензировала большинство своих технологий Java под лицензией GPL-2.0-only. Oracle предлагает свою собственную виртуальную машину HotSpot Java Virtual Machine, однако официальной эталонной реализацией является OpenJDK JVM, которая является бесплатным программным обеспечением с открытым исходным кодом, используется большинством разработчиков и является JVM по умолчанию почти для всех дистрибутивов Linux.

История

Джеймс Гослинг, Майк Шеридан и Патрик Нотон начали проект по созданию языка Java в июне 1991 года. Java изначально была разработана для интерактивного телевидения, но в то время она была слишком сложной для индустрии цифрового кабельного телевидения. Первоначально язык назывался Oak в честь дуба, который стоял возле офиса Гослинга. Позже проект получил название Green и, наконец, был переименован в Java, от Java coffee, сорта кофе из Индонезии.Гослинг разработал Java с синтаксисом в стиле C/C++, который будет знаком системным и прикладным программистам.

Принципы

При создании языка Java преследовалось пять основных целей:

Он должен быть простым, объектно-ориентированным и знакомым.

Он должен быть надежным и безопасным.

Он должен быть нейтральным к архитектуре и переносимым.

Он должен выполняться с высокой производительностью.

Он должен быть интерпретируемым, потоковым и динамическим.

Синтаксис

Синтаксис Java в значительной степени сформировался под влиянием C++ и C. В отличие от C++, который сочетает в себе синтаксис структурного, общего и объектно-ориентированного программирования, Java был создан почти исключительно как объектно-ориентированный язык. Весь код пишется внутри классов, и каждый элемент данных является объектом, за исключением примитивных типов данных (т.е. целых чисел, чисел с плавающей точкой, булевых значений и символов), которые не являются объектами по причинам производительности. Java повторно использует некоторые популярные аспекты C++ (например, метод printf).

В отличие от C++, Java не поддерживает перегрузку операторов и множественное наследование для классов, хотя множественное наследование поддерживается для интерфейсов.

В Java используются комментарии, аналогичные комментариям в C++. Существует три различных стиля комментариев: однострочный, обозначаемый двумя косыми чертами (//), многострочный, открываемый символом /* и закрываемый символом */, и стиль комментариев Javadoc, открываемый символом /** и закрываемый символом */. Стиль комментирования Javadoc позволяет пользователю запустить исполняемый файл Javadoc для создания документации к программе и может быть прочитан некоторыми интегрированными средами разработки (IDE), такими как Eclipse, чтобы позволить разработчикам получить доступ к документации в IDE. Все исходные файлы должны быть названы по имени публичного класса, который они содержат, с добавлением суффикса .java, например, HelloWorldApp.java. Сначала он должен быть скомпилирован в байткод с помощью компилятора Java, в результате чего создается файл с суффиксом .class (в данном случае HelloWorldApp.class). Только после этого он может быть выполнен или запущен. Исходный файл Java может содержать только один открытый класс, но он может содержать несколько классов с модификатором непубличного доступа и любое количество открытых внутренних классов. Когда исходный файл содержит несколько классов, необходимо сделать один класс (представленный ключевым словом class) публичным (которому предшествует ключевое слово public) и назвать исходный файл именем этого публичного класса.

Класс, который не объявлен как public, может храниться в любом файле .java. Компилятор создаст файл класса для каждого класса, определенного в исходном файле. Имя файла класса - это имя класса с добавлением .class. При генерации файла класса анонимные классы обрабатываются так, как если бы их имя было конкатенацией имени их объемлющего класса, $ и целого числа.

<Ключевое слово public означает, что метод может быть вызван из кода других классов, или что класс может быть использован классами вне иерархии классов. Иерархия классов связана с именем каталога, в котором находится файл .java. Это называется модификатором уровня доступа. Другие модификаторы уровня доступа включают ключевые слова private (метод, доступ к которому возможен только в том же классе) и protected (разрешает доступ коду из того же пакета). Если часть кода пытается получить доступ к приватным или защищенным методам, JVM выбросит SecurityException

Ключевое слово static перед методом означает статический метод, который связан только с классом, а не с каким-либо конкретным экземпляром этого класса. Только статические методы могут быть вызваны без ссылки на объект. Статические методы не могут обращаться к членам класса, которые также не являются статическими. Методы, не обозначенные как статические, являются методами экземпляра и требуют для работы конкретного экземпляра класса.

Ключевое слово void указывает на то, что метод main не возвращает никакого значения вызывающей стороне. Если программа на Java должна завершиться с кодом ошибки, она должна явно вызвать System.exit().

Имя метода main не является ключевым словом в языке Java. Это просто имя метода, который вызывает программа запуска Java для передачи управления программе. Классы Java, работающие в управляемых средах, такие как апплеты и Enterprise JavaBeans, не используют и не нуждаются в методе main(). Программа Java может содержать несколько классов, имеющих методы main, что означает, что виртуальной машине необходимо явно указать, из какого класса запускать программу.

Метод main должен принимать массив объектов String. По традиции он обозначается как args, хотя можно использовать любое другое законное имя идентификатора. Начиная с Java 5, метод main может также использовать переменные аргументы в форме public static void main(String... args), что позволяет вызывать метод main с произвольным количеством аргументов String. Эффект этого альтернативного объявления семантически идентичен (параметр args по-прежнему является массивом объектов String), но он позволяет использовать альтернативный синтаксис для создания и передачи массива.

Пусковая установка Java запускает Java, загружая заданный класс (указанный в командной строке или как атрибут в JAR) и запуская его метод public static void main(String[]). Автономные программы должны объявить этот метод явно. Параметр String[] args - это массив объектов String, содержащий любые аргументы, переданные классу. Параметры в main часто передаются с помощью командной строки.

Печать является частью стандартной библиотеки Java: Класс System определяет публичное статическое поле, называемое out. Объект out является экземпляром класса PrintStream и предоставляет множество методов для печати данных на стандартный вывод, включая println(String), который также добавляет новую строку к переданной строке.

Строка "Hello World!" автоматически преобразуется компилятором в объект String.
